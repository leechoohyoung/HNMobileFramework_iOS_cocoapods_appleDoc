Overview

NSDictionary 의 구조를 변환하는 클래스

내부적으로 재귀함수를 호출하는 구조이므로 중첩 횟수가 너무 커지면 stack over flow 발생할 수 있으므로 너무 깊은 자료구조를 변환하지 않도록 주의할 것

Discussion

각 하위 NSDictionary 들은 구두점을 기준으로 연결됨.

예를 들어,

NSDictionary *nestedDic = @{@“key0”:@“val0”, @“key1”:@{@“subKey0”:@“subVal0”, @“subKey1”:@{@“subSubKey0”:@“subSubVal0”}}, @“key2”:@“val2”, @“key3”:@{@“subKey0”:@“subVal0”, @“subKey1”:@“subVal1”, @“subKey2”:@“subVal2”}};

위의 NSDictionary 는 아래와 같이 변형됨.

{ “key0” = val0; “key1.subKey0” = subVal0; “key1.subKey1.subSubKey0” = subSubVal0; “key2” = val2; “key3.subKey0” = subVal0; “key3.subKey1” = subVal1; “key3.subKey2” = subVal2; }

깊이 우선 탐색으로 작업을 수행함.

callBack 객체의 각 블럭은 main thread 에서 호출됨.
